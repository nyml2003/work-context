# 状态与错误模型

## 状态建模重点

- 是否存在 loading、success、empty、error 多状态
- 不同状态是否需要不同字段集合
- 重试、恢复、局部更新是否会改变状态机

## 默认规则

1. UI 依赖显式状态差异时，优先使用 discriminated union。
2. 错误模型要说明是用户可见错误、技术失败还是诊断信息。
3. 不要混用 `undefined`、`null`、空字符串和魔法值表达“无值”。
4. 状态转移要可读，不把状态语义埋进多个布尔字段的组合里。

## 输出给 worker 时至少说明

- 状态枚举或联合结构
- 错误字段形态
- 可恢复路径和重试行为
- 是否需要把临时状态拆到局部组件内
