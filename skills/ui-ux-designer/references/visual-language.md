# Visual Language

## 适用时机

- 需要决定界面调性、CSS 样式表现、视觉层级和组件外观时。

## 关注点

1. 用色是否建立了清晰的主次，而不是堆更多颜色。
2. 排版是否稳定：字号层级、字重、行高、字距和段落节奏是否一致。
3. 留白和密度是否匹配任务强度；信息密度高不等于拥挤。
4. 对比是否足够，让标题、正文、辅助信息、危险操作一眼可分。
5. CSS 表现是否服务结构，而不是靠阴影、描边、渐变堆出“设计感”。
6. 组件外观是否一致：按钮、输入框、卡片、标签、导航要有明确家族关系。

## 输出

- 一份视觉方向说明，描述主色调、层级、密度和氛围。
- 一份主要视觉问题清单，说明哪些地方削弱了理解或信任感。
- 一份 CSS 样式优化建议，说明哪些属性或表现方式需要收敛。

## 反模式

- 所有层级都想抢注意力。
- 用更多装饰替代更清晰的信息层级。
- CSS 层面堆过多细碎效果，导致界面发虚、发脏或不稳定。
