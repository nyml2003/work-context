# Design Discovery

## 适用时机

- 新产品、新页面、新流程从 0 到 1。
- 现有界面要改版，但目标、用户或问题还没有被说清。
- 需要先判断当前问题到底是视觉问题、交互问题，还是动线问题。

## 关注点

1. 明确产品目标、业务目标和用户目标，不要把三者混为一谈。
2. 识别主要用户、次级用户、核心任务和高频场景。
3. 明确当前媒介与环境：Web、移动端、桌面端、大屏、弱网、触屏或键鼠。
4. 识别成功标准，例如理解速度、完成率、转化率、操作时长、错误率或满意度。
5. 明确本次设计的边界和不做项，避免把所有问题都卷进一次改版。

## 输出

- 一份问题定义，说明要解决什么、不解决什么。
- 一份用户与任务说明，说明谁在什么场景下完成什么动作。
- 一份优先级判断，说明先解决哪个体验问题最有价值。

## 反模式

- 只有风格偏好，没有用户任务。
- 只讨论“好不好看”，不讨论是否更好理解和更好操作。
- 在目标不清的情况下直接定视觉稿。
