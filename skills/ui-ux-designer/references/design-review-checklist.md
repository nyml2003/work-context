# Design Review Checklist

## 适用时机

- 任何界面方案评审、改版验收、交互优化或体验排查。

## 默认检查项

1. 视觉层级是否清晰，用户能否在几秒内分辨主任务、主信息和辅助信息。
2. CSS 样式是否稳定、统一、克制，没有无意义的装饰噪音。
3. 布局是否支持主要任务，模块关系、间距、对齐和响应式是否可靠。
4. 交互是否可预测，状态反馈、错误恢复、焦点和禁用逻辑是否完整。
5. 动画是否帮助理解层级和状态，而不是拖慢操作或制造干扰。
6. 动线是否顺畅，用户是否知道当前所处位置、下一步和回退路径。
7. 可访问性是否达标，包括对比度、键盘可达、可点击区域、语义与动效克制。

## 输出格式

- 先写目标与主要问题。
- 再按严重程度列出设计问题和理由。
- 最后给出改进方向、优先级和剩余风险。

## 通过标准

- 主要任务路径清晰，没有明显理解障碍。
- 视觉、布局、交互、动画和动线互相支持，而不是互相打架。
- 不依赖额外解释，用户也能理解页面结构和下一步操作。
