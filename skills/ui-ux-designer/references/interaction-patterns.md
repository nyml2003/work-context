# Interaction Patterns

## 适用时机

- 需要判断操作方式、状态反馈、表单交互、导航逻辑和错误恢复时。

## 关注点

1. 操作入口是否明显，命名是否符合用户心智。
2. 状态反馈是否及时，用户是否知道“点击后发生了什么”。
3. 表单、列表、筛选、弹层、菜单、分页等交互模式是否符合常见预期。
4. 错误提示是否能帮助恢复，而不是只报错。
5. 是否减少不必要的确认、跳转和来回切换。
6. 键盘、焦点、触屏区域、悬停态、禁用态是否清楚且可操作。

## 输出

- 一份交互问题清单，说明阻塞、迷惑、冗余或高成本步骤。
- 一份改进建议，说明如何让操作更直接、更可预测。
- 一份状态设计建议，覆盖默认、悬停、激活、加载、成功、失败、空态等。

## 反模式

- 靠用户猜测下一步。
- 状态变化无反馈，或反馈过多导致噪音。
- 把异常路径完全留给用户自己兜底。
