# 组件边界

## 拆分时优先看

- 这个组件负责展示、协调还是状态承载
- props 是否表达了单一意图
- 是否存在可以独立复用的叶子结构
- 是否有明显应该抽出的条件分支

## 默认规则

1. 页面壳、业务容器、叶子组件尽量不要混成一层。
2. 一个组件的 props 应围绕一个清晰职责组织。
3. 临时 helper 可以存在，但不能替代真正的组件边界。
4. 如果一个组件既决定业务状态又负责复杂视觉结构，默认过重。

## 反模式

- `props` 只是把上游所有变量原样透传。
- 一个组件里同时完成数据整理、状态切换和大段复杂 JSX。
- 叶子组件反向感知过多上层业务语义。
