# 类型边界

## 需要先定义的边界

- 外部输入如何进入系统
- UI 层实际消费什么结构
- 哪些类型是共享域模型，哪些只是局部实现细节
- 哪些地方需要显式 narrowing

## 默认规则

1. 网络响应、浏览器输入、URL 参数、存储恢复值都视为不可信输入。
2. 不要把原始 transport 类型直接泄露给深层 UI。
3. 尽量用命名类型表达稳定边界，不要到处复制匿名对象字面量。
4. 能在边界处收窄的，不把 `unknown` 或宽泛联合拖进核心逻辑层。

## 常见反模式

- 用 `any` 抹平边界不确定性。
- 同一个概念在不同文件里各自定义近似类型。
- 为了图省事把接口响应 shape 直接传进组件树。
