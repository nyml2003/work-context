# 依赖分类

## 依赖类型

1. 信息依赖：下游只需要一份稳定摘要，例如 TypeScript 约束、日志约束。
2. 结构依赖：下游必须等待上游真实代码或结构落地，例如共享 state shape、组件拆分结果。
3. 验证依赖：review 或测试需要等待 worker 产出后才能开始。

## 默认判断

- 信息依赖通常可以通过 handoff 解耦。
- 结构依赖通常需要串行。
- 验证依赖一般位于链路末端。

## 使用方式

- 先找出真正阻塞代码落地的依赖。
- 不要把所有“最好先知道”的信息都当成强依赖。
- 能靠 policy 摘要解决的，就不要让每个 worker 重建一遍上下文。
