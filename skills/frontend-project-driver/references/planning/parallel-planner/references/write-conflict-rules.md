# 写入冲突规则

## 默认冲突点

- 同一组件树或页面骨架
- 同一组共享类型
- 同一路由或导航结构
- 同一接口契约与映射层
- 同一日志工具或埋点 schema

## 判断规则

1. 如果两个子任务主要修改同一文件族，默认冲突。
2. 如果两个子任务虽然改不同文件，但依赖同一份尚未稳定的决策，也视为冲突。
3. 如果一个子任务只是消费上游稳定 handoff，不直接改上游 ownership，可以并行。

## 反模式

- 让两个 worker 同时改同一个页面主文件。
- 在共享 state shape 未定时并行推进多个下游实现。
- 把“并行”建立在口头约定而不是明确 handoff 上。
