# Layout And Composition

## 适用时机

- 需要判断页面骨架、布局栅格、信息编排和响应式结构时。

## 关注点

1. 页面是否有明确骨架：导航区、主要内容区、辅助区、操作区。
2. 版式是否支持主任务，还是让次要信息抢占了首屏。
3. 栅格、对齐、间距和分组是否稳定，避免“看起来差不多”的松散布局。
4. 组件之间是否形成阅读节奏：先看哪里，再看哪里，是否自然。
5. 响应式或跨端切换时，布局是否保持层级清晰，而不是只做机械压缩。
6. 长列表、表单、详情页、仪表盘等不同视图是否采用合适的布局策略。

## 输出

- 一份布局判断，说明当前视图的主次结构是否成立。
- 一份版式优化建议，说明哪些模块要合并、拆分、前置或后置。
- 一份跨尺寸适配建议，说明布局在不同媒介下如何保持可读性。

## 反模式

- 所有区域权重接近，用户不知道先看哪里。
- 只会堆卡片或堆分栏，没有明确结构逻辑。
- 响应式只改宽度，不改信息优先级。
