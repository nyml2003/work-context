# User Flow And Journeys

## 适用时机

- 需要分析任务路径、转化路径、漏斗、跨页面操作链路和认知负担时。

## 关注点

1. 用户从进入页面到完成任务，是否有连续、清晰、低阻塞的动线。
2. 首屏是否给出正确的下一步，而不是把用户扔进信息堆里。
3. 关键路径上的每一步是否必要，是否存在可合并或可提前的信息。
4. 用户能否随时知道自己当前在哪一步、下一步去哪、能否返回。
5. 跨页面、跨模块、跨设备切换时，信息是否衔接自然。
6. 动线设计是否兼顾新手学习成本和熟练用户效率。

## 输出

- 一份动线判断，说明主路径是否顺畅、哪里最容易断。
- 一份路径优化建议，说明哪些步骤该前置、后移、合并或删除。
- 一份关键转化或关键任务视角的优先级建议。

## 反模式

- 页面内部还算整齐，但整个任务路径断裂。
- 把重要操作藏在次级区域或过深层级。
- 只优化局部组件，不优化整条任务链路。
