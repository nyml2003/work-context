# Motion And Animation

## 适用时机

- 需要设计或评审进入退出、状态过渡、层级切换和微动效时。

## 关注点

1. 每个动画都要先回答“帮助了什么理解或操作”。
2. 动画节奏要符合界面重量：轻交互轻反馈，重切换重过渡。
3. 层级切换、弹层开合、列表更新、页面转场要帮助用户建立因果关系。
4. 动画时长、缓动和频率要克制，避免界面显得拖沓或花哨。
5. 多个动效同时出现时，要有主次，避免争抢注意力。
6. 注意降低晕动和分散注意力的风险，并考虑减少动画的可用性需求。

## 输出

- 一份动画目的说明，说明哪些动效保留、哪些取消、哪些减弱。
- 一份关键动效建议，描述触发点、节奏和预期感受。
- 一份风险说明，指出哪些动画会破坏效率、稳定感或可访问性。

## 反模式

- 先加动效，再找理由。
- 所有元素都在动，结果没有重点。
- 用动画掩盖布局和交互本身的不清晰。
