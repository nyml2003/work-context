# 实施流程

## 优先顺序

1. 页面或容器壳体
2. 主任务区域与信息层级
3. loading、empty、error 等状态位
4. 交互反馈和细节 polish

## 默认规则

- 先让结构稳定，再做视觉微调。
- 组件拆分围绕可读性和 ownership，而不是为了“看起来更组件化”。
- 任何状态位都要有落点，不只实现 happy path。
- 尽量把 render-ready 数据传入 UI，不让 UI 重新推导复杂逻辑。

## 反模式

- 一个文件里同时塞页面布局、接口映射和复杂 state 逻辑。
- 页面已经能跑，但没有 empty 或 error 状态。
- 把样式问题和逻辑问题混在同一次重构里。
